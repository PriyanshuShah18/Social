================================================================================
  SOCIAL POSTING AGENT — REQUIREMENTS & SETUP GUIDE
  What you need, where to get it, and what to do per platform
================================================================================

This file lists every external dependency, credential, and service required to
build and run the Social Posting Agent. Follow each section in order.

================================================================================
  2. MONGODB — INSTALL PER PLATFORM
================================================================================

  ── WINDOWS ──
  Option A (recommended): MongoDB Community Server
    1. Download from: https://www.mongodb.com/try/download/community
    2. Run the MSI installer.
    3. Choose "Complete" installation.
    4. Check "Install MongoDB as a Service" (starts automatically).
    5. MongoDB Compass (GUI) is included — use it to inspect your data.
    6. Verify:
         mongosh
         use socialagent

  Option B: Docker
       docker run -d --name mongodb -p 27017:27017 mongo:7

  ── macOS ──
  1. Install via Homebrew:
       brew tap mongodb/brew
       brew install mongodb-community@7.0
       brew services start mongodb-community@7.0
  2. Verify:
       mongosh
       use socialagent

  ── LINUX (Ubuntu/Debian) ──
  1. Follow official instructions at:
       https://www.mongodb.com/docs/manual/tutorial/install-mongodb-on-ubuntu/
  2. Summary:
       curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | sudo gpg --dearmor -o /usr/share/keyrings/mongodb-server-7.0.gpg
       echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
       sudo apt update
       sudo apt install -y mongodb-org
       sudo systemctl start mongod
       sudo systemctl enable mongod
  3. Verify:
       mongosh
       use socialagent

  ── MONGODB ATLAS (cloud — no install needed) ──
  If you don't want to install MongoDB locally:
    1. Go to: https://www.mongodb.com/cloud/atlas/register
    2. Create a free M0 cluster (free tier, no credit card needed).
    3. Click "Connect" → "Drivers" → copy the connection string.
    4. Whitelist your IP (or use 0.0.0.0/0 for dev).
    5. Your connection string will look like:
         mongodb+srv://user:password@cluster0.xxxxx.mongodb.net/socialagent?retryWrites=true&w=majority

  Your MONGODB_URI will be one of:
    mongodb://localhost:27017/socialagent           (local)
    mongodb+srv://user:pass@cluster.xxx.net/socialagent  (Atlas)


================================================================================
  3. LINKEDIN DEVELOPER APP
================================================================================

  WHAT YOU NEED:
    - LINKEDIN_CLIENT_ID
    - LINKEDIN_CLIENT_SECRET
    - A LinkedIn Company Page (for org posting)

  WHERE TO GET IT:
    https://www.linkedin.com/developers/apps

  EXACT STEPS:

  1. Go to https://www.linkedin.com/developers/apps and click "Create App".

  2. Fill in:
     - App name: e.g. "Social Posting Agent"
     - LinkedIn Page: Select your company page (required)
       → If you don't have one, create it first at:
         https://www.linkedin.com/company/setup/new/
     - App logo: Upload any logo
     - Legal agreement: Accept

  3. After creation, go to the "Auth" tab:
     - Copy the Client ID      → LINKEDIN_CLIENT_ID
     - Copy the Client Secret  → LINKEDIN_CLIENT_SECRET
     - Add redirect URL:
         http://localhost:3000/api/auth/linkedin/callback

  4. Go to the "Products" tab and request access to:
     - "Share on LinkedIn"          → gives w_member_social
     - "Advertising API" (optional) → gives w_organization_social
     NOTE: w_organization_social may require app review.

  5. Go to the "Settings" tab:
     - Note the API version format: YYYYMM (e.g., 202606)
     - Set LINKEDIN_API_VERSION=202606

  SCOPES NEEDED:
    - w_member_social           — for personal profile posting
    - w_organization_social     — for company page posting
    - r_basicprofile (or openid profile email) — to identify the member

  COMPANY PAGE ROLES (for org posting):
    The authenticated member must have one of these roles on the Company Page:
    - ADMINISTRATOR
    - DIRECT_SPONSORED_CONTENT_POSTER
    - CONTENT_ADMIN
    To assign roles: Company Page → Admin tools → Manage admins

  LINKEDIN API HEADERS (required on every API call):
    Linkedin-Version: 202606
    X-Restli-Protocol-Version: 2.0.0

  DOCUMENTATION:
    - Posts API: https://learn.microsoft.com/en-us/linkedin/marketing/community-management/shares/posts-api
    - OAuth:    https://learn.microsoft.com/en-us/linkedin/shared/authentication/authorization-code-flow

  4. META (FACEBOOK + INSTAGRAM) DEVELOPER APP
================================================================================

  WHAT YOU NEED:
    - META_APP_ID
    - META_APP_SECRET
    - A Facebook Page you administer
    - An Instagram Professional account linked to that Facebook Page

  WHERE TO GET IT:
    https://developers.facebook.com/apps/

  EXACT STEPS:

  1. Go to https://developers.facebook.com/ and log in.

  2. Click "My Apps" → "Create App".
     - Use case: "Other" or "Business"
     - App type: "Business"
     - App name: "Social Posting Agent"
     - Contact email: your email

  3. After creation, go to "App Settings" → "Basic":
     - Copy App ID       → META_APP_ID
     - Copy App Secret   → META_APP_SECRET

  4. Add Products:
     - Click "Add Product" in the left sidebar
     - Add "Facebook Login for Business" (or "Facebook Login")
     - In Facebook Login Settings, add the redirect URI:
         http://localhost:3000/api/auth/meta/callback

  5. Set the Graph API version:
     - META_GRAPH_VERSION=v25.0
     - Check the latest at: https://developers.facebook.com/docs/graph-api/changelog

  PERMISSIONS NEEDED:
  ┌─────────────────────────────────────┬───────────────────────────────────┐
  │ Permission                          │ Purpose                           │
  ├─────────────────────────────────────┼───────────────────────────────────┤
  │ pages_show_list                     │ List pages the user manages       │
  │ pages_read_engagement               │ Read page post engagement         │
  │ pages_manage_posts                  │ Create and manage posts on Pages  │
  │ instagram_basic                     │ Read IG account info              │
  │ instagram_content_publish           │ Publish to Instagram              │
  │ instagram_manage_comments (opt.)    │ Manage IG comments                │
  └─────────────────────────────────────┴───────────────────────────────────┘

  IMPORTANT NOTES:
  - In DEVELOPMENT mode, only users with roles on the app (Admin, Developer,
    Tester) can use the app. To let others use it → go through App Review.
  - Add yourself as a Tester at: App Roles → Roles → Add Testers.

  ── FACEBOOK PAGE SETUP ──
  - You need a Facebook Page to post to.
  - Create one at: https://www.facebook.com/pages/create
  - You must be an Admin of the Page.

  ── INSTAGRAM PROFESSIONAL ACCOUNT SETUP ──
  - Go to your Instagram app → Settings → Account → Switch to Professional
  - Choose "Business" or "Creator"
  - Link it to your Facebook Page:
      Instagram → Settings → Account → Linked Accounts → Facebook → select your Page
  - CRITICAL: Instagram publishing ONLY works with Professional accounts.
    Personal accounts cannot publish via API.
  - The image_url used in Instagram container creation must be a PUBLICLY
    ACCESSIBLE URL (not localhost). Use S3 or a CDN.

  DOCUMENTATION:
    - Facebook Pages API:       https://developers.facebook.com/docs/pages-api
    - Instagram Content Publish: https://developers.facebook.com/docs/instagram-platform/instagram-api-with-instagram-login/content-publishing
    - Graph API Explorer (testing): https://developers.facebook.com/tools/explorer/

  5. FILE STORAGE (S3 OR COMPATIBLE)
================================================================================

  WHAT YOU NEED:
    - A storage bucket for uploaded media (images/videos)
    - Public read access (Instagram requires publicly reachable image URLs)

  OPTIONS:

  ── AWS S3 ──
  1. Sign up at: https://aws.amazon.com
  2. Go to S3 Console → Create Bucket
  3. Bucket name: e.g., "social-agent-media"
  4. Unblock public access (for media that Instagram needs to read)
  5. Create an IAM user with S3 permissions
  6. Get Access Key ID and Secret Access Key
  Env vars:
    AWS_ACCESS_KEY_ID=...
    AWS_SECRET_ACCESS_KEY=...
    AWS_S3_BUCKET=social-agent-media
    AWS_S3_REGION=us-east-1

  ── Cloudflare R2 (S3-compatible, free tier generous) ──
  1. Sign up at: https://dash.cloudflare.com
  2. Go to R2 → Create Bucket
  3. Create an API token with R2 read/write permissions
  Env vars:
    AWS_ACCESS_KEY_ID=...  (R2 API token ID)
    AWS_SECRET_ACCESS_KEY=... (R2 API token secret)
    AWS_S3_BUCKET=social-agent-media
    AWS_S3_ENDPOINT=https://<account-id>.r2.cloudflarestorage.com

  ── MinIO (self-hosted, free, for local dev) ──
  1. Docker:
       docker run -d -p 9000:9000 -p 9001:9001 \
         --name minio \
         -e MINIO_ROOT_USER=minioadmin \
         -e MINIO_ROOT_PASSWORD=minioadmin \
         minio/minio server /data --console-address ":9001"
  2. Open http://localhost:9001, create a bucket
  Env vars:
    AWS_ACCESS_KEY_ID=minioadmin
    AWS_SECRET_ACCESS_KEY=minioadmin
    AWS_S3_BUCKET=social-agent-media
    AWS_S3_ENDPOINT=http://localhost:9000

  NOTE: For Instagram, media must be at a PUBLIC URL. During local dev, you
  can use ngrok or a similar tunnel, or upload to a real cloud bucket.


================================================================================
  6. ENCRYPTION KEY
================================================================================

  WHAT:
    A 32-byte random key for AES-256-GCM encryption of OAuth tokens at rest.

  HOW TO GENERATE:

  ── Node.js (any platform) ──
    node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

  ── Linux/macOS ──
    openssl rand -hex 32

  ── PowerShell (Windows) ──
    [Convert]::ToHexString([System.Security.Cryptography.RandomNumberGenerator]::GetBytes(32))

  Set in .env:
    ENCRYPTION_KEY=<your-64-char-hex-string>

================================================================================
  9. PLATFORM-SPECIFIC GOTCHAS & TIMELINES
================================================================================

  LINKEDIN:
  - App review can take 2-4 weeks for w_organization_social.
  - w_member_social is usually instant for the app creator.
  - Test with personal posting first (no review needed).
  - LinkedIn rate limits: ~100 posts/day per member.
  - The Posts API replaced the legacy UGC Post API.

  FACEBOOK:
  - In dev mode, only app role holders can use the app.
  - App Review is required for public use of pages_manage_posts.
  - App Review can take 1-5 business days.
  - You must provide a screencast demo during review.
  - Page tokens don't expire if derived from a long-lived user token.

  INSTAGRAM:
  - Publishing ONLY works with Professional (Business/Creator) accounts.
  - Image URLs must be publicly accessible (not localhost).
  - Video/reel publishing: container goes through a processing state;
    you must poll GET /{container-id}?fields=status_code until FINISHED.
  - Rate limit: 25 API-published posts per 24 hours per IG account.
  - Carousel posts: create child containers first, then a carousel container.

  GENERAL:
  - Never store tokens in plain text in the database.
  - Meta requires Business Verification for advanced permissions.
    Do this at: https://business.facebook.com/settings/info
  - All platforms may reject posts containing certain URLs or content.
  - Always handle token expiry and implement refresh flows.
